## 2025-01-20 - Optimize string parsing in loops for bash scripts
**Learning:** Spawning subprocesses (`grep`, `awk`, `head`) inside loops in shell scripts severely degrades performance. Using native bash regex matching (`=~`) on strings already in memory avoids subprocess overhead. However, on macOS (which uses Bash 3.2), inline regex with spaces or special characters in `[[ ... ]]` causes errors, so patterns must be assigned to variables first (`re="..."`). Also, `^` matches the start of the entire string, not the start of a line, so `([[:space:]]|^)` should be used.
**Action:** Replace `grep`/`awk` pipelines inside loops with native bash regex (`=~`) when checking variables already in memory. Store the regex in a variable before using it in `[[ "$var" =~ $re ]]` to ensure cross-platform/macOS compatibility.
