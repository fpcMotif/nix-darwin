## 2024-06-20 - Subprocess Overhead in Bash String Parsing
**Learning:** Parsing strings already in memory using `grep`, `head`, and `awk` in Bash loops creates significant subprocess overhead. In `verify-macos-settings.sh`, looping through 9 power management settings using pipeline subprocesses (`printf '%s' "$pm" | grep ... | head -1 | awk ...`) caused a noticeable performance bottleneck compared to native Bash matching.
**Action:** For strings already loaded into variables, use native Bash regex (`[[ "$var" =~ $re ]]`) to match lines and extract fields (`${BASH_REMATCH[2]}`). To handle multiline matches in macOS Bash 3.2, use `([[:space:]]|^)` instead of `^` to match the beginning of a line. This pattern reduces the lookup from spawning two external processes per iteration down to a fast internal evaluation.
