## 2024-06-11 - Optimize pmset bash script loop
**Learning:** In Bash scripts, replacing compiled C binaries (like `grep` or `awk`) inside loops with native Bash regex (`=~`) results in a significant performance improvement (e.g. 5s -> 0.07s for 100 iterations) when searching strings loaded in memory. However, to maintain macOS compatibility (Bash 3.2), inline regular expressions inside `[[ ... ]]` are poorly supported and the regex pattern must be assigned to a separate variable first (e.g., `re=$'(^|\n)[[:space:]]*'"$key"'[[:space:]]+([^[:space:]]+)'; [[ "$pm" =~ $re ]]`).
**Action:** Avoid spawning subprocesses like `grep` or `awk` inside loops when processing multiline text already in memory. Instead, use native Bash regex matching with a pre-assigned variable for cross-platform compatibility.
