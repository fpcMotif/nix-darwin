## 2026-06-06 - Bash subprocess overhead in update scripts
**Learning:** Shell scripts frequently called during CI/CD (like those in `scripts/lib/auto-update.sh`) suffer significant performance degradation due to forking subprocesses (`grep`, `awk`, `head`, `cut`) inside loops or string parsing tasks. A standard pipeline like `printf | grep | head | sed` can take ~286ms for 100 iterations, whereas native bash regex takes ~49ms.
**Action:** When extracting substrings from small files or known variables in bash scripts, avoid pipelines with external binaries. Instead, use native Bash regex matching (`[[ "$var" =~ $re ]]`) and capture groups (`${BASH_REMATCH[1]}`). To ensure macOS compatibility, always assign the regex to a variable first (`local re='pattern'`). Ensure you clean up all test/scratchpad scripts from the workspace before committing.
